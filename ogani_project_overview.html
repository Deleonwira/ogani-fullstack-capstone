Memahami Project
Ogani Fullstack

Panduan visual dan interaktif untuk memahami bagaimana ketiga komponen (Mobile App, Backend API, dan Admin Web) bekerja sama membentuk satu kesatuan sistem *e-commerce*.

1. Gambaran Besar (Analogi Restoran)

Untuk mempermudah pemahaman, bayangkan sistem aplikasi ini seperti sebuah Restoran Cepat Saji yang beroperasi.

smartphone

1. Buku Menu / Pelayan

Mobile App (Flutter)

Aplikasi yang dipegang oleh pelanggan. Di sinilah pelanggan melihat daftar makanan (produk), memasukkan ke keranjang, dan menekan tombol "Pesan". Aplikasi ini tidak membuat makanannya sendiri, ia hanya mencatat pesanan.

dns

2. Dapur Restoran

Backend API (Spring Boot)

Dapur pusat tempat semua pesanan diproses. Ketika pelanggan memesan lewat HP, dapur menerimanya, memasaknya, mengurangi stok bahan baku di kulkas (Database), dan mengirim kembali status "Makanan Siap".

dashboard

3. Ruang Manajer

Admin Web (CodeIgniter)

Tempat manajer restoran memantau segalanya. Manajer tidak memasak, ia melihat laporan keuangan, mengubah harga menu, melihat daftar pelanggan, dan memantau status pesanan harian.

Catatan Penting: Dalam proyek ini, Flutter App berinteraksi penuh dengan Spring Boot API menggunakan format JSON. Sedangkan Admin Web berinteraksi secara langsung dengan MySQL Database menggunakan Model CodeIgniter 4 (MVC pattern).

3. Contoh Proses Pengerjaan Fitur

1

Perancangan Database (MySQL)

Setiap fitur dimulai dari struktur data. Jika ada fitur baru (misal: "Promo"), kita harus memastikan tabel `promos` tersedia di MySQL terlebih dahulu.

2

Membangun API (Spring Boot)

Backend Developer membuat *endpoint* API (contoh: `GET /api/promos`). File yang diubah: `Promo.java` (Model), `PromoRepository.java`, `PromoService.java`, dan `PromoController.java`.

3

Menghubungkan Aplikasi (Flutter)

Mobile Developer memanggil API tadi menggunakan HTTP. File yang dibuat: `promo_service.dart` (untuk mengambil data JSON dari backend), dan `promos_screen.dart` (untuk menampilkan UI ke pengguna).

4

Mengatur via Admin Panel (CodeIgniter)

Admin Developer membuat halaman `Views/admin/promos.php` dan `Controllers/Admin/Promo.php` agar Manajer bisa menambah, mengubah, dan menghapus Promo langsung dari *browser*.
